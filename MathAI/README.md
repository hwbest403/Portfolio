**학생은 파이썬 풀이를 얻고자 하는 문제를 시스템에 입력한다.
*보조액터 :
**1) 문제 풀이/답 제공 시스템
***문제 풀이/답 제공 시스템은 Math Problem Solver Model이 생성한 파이썬 코드 풀이 와 답을 학생에게 제공한다.
**2) 교사
***교사는 파이썬 코드를 얻고자 하는 문제를 시스템에 입력한다.
**3) 시스템 관리자
***문제 풀이 된 데이터를 추가한다.
**4) 문제 풀이 관리자
***문제 풀이를 추가한다.

==== 선행 조건 ====
*사용자는 수학 문제를 제공한 상태이어야 한다.
*문제에 대한 전처리 및 유형분석이 되어 있어야 한다.

==== 후행 조건 ====
*생성된 Equation tree가 binary tree이어야 한다. (child의 개수가 2개이어야 한다.)

==== 시나리오 ====
*기본 시나리오
**1) '''사용자'''는 문제 입력란에 <u>수학 문제 텍스트</u>를 입력한다.
**2) 시스템은 입력된 문제 텍스트에 대하여 전처리를 진행한다.
**3) 시스템은 전처리가 진행된 문제에 대하여 유형 분석을 진행하여 레이블링한다.
**4) 시스템은 전처리와 유형분석이 완료된 문제에 대하여 Equation Tree를 생성한다.
**5) 시스템은 생성된 Equation Tree와 사전 정의된 함수 파일을 기반으로 파이썬 코드 풀이를 생성한다.
**6) 시스템은 파이썬 코드 풀이를 실행한다.
**7) 시스템은 <u>파이썬 코드와 답</u>을 문제 '''풀이/답 제공 시스템'''에 반환한다.
**8) '''문제 풀이/답 제공 시스템'''은 반환된 <u>풀이와 답</u>을 '''사용자'''에게 제공한다.
*대안 시나리오
**'''A1: 빈 텍스트가 입력되었을 시'''
***(1)에서 빈 텍스트가 입력되었을 때
***1) 에러 문구를 출력한 후 시스템을 동작시키지 않는다.

**'''A2: 생성된 Equation Tree가 binary tree가 아닐 시'''
***(5)에서 비정상적으로 Equation Tree를 생성했을 때
***1) child node의 개수가 한 개 이하이면 부족한 파라미터 수를 none으로 채워 넣는다.
***2) child node의 개수가 3개 이상이면 첫 2개 파라미터만 남기고 뒤의 노드를 삭제한다.

== 프로젝트 모델 ==
* Graph2Tree 모델
저희 팀의 모델은 풀이가 사칙연산으로만 이루어져 있는 영어 서술형 수학 문제 데이터셋에 대하여 뛰어난 성능을 보인 Graph2Tree모델을 기반으로 모델을 구축하였습니다.<br><br>
[[파일:image.png|500픽셀]]<br><br>
Graph2Tree 모델은 관계성과 순서정보를 두 개의 그래프, Quantity Cell Graph와 Quantity Comparison Graph를 사용하여 MWP 문제 내의 quantity를효과적으로 표현합니다.<br>
Quantity Cell Graph는 quantity와문장 내 중요한 단어들을 연관 짓는 그래프, 
Quantity Comparison Graph는 quantity 간에 관계를 나타내는 그래프입니다.<br>
또한 그래프 기반의 인코더와 트리 기반의디코더를 활용하여 더 나은 풀이 표현을 생성합니다.<br>
* KoGraph2Tree 모델
KoGraph2Tree 모델은 기본적인 사칙연산뿐만 아니라 일반적인 서술형 수학 문제에서 사용하는 풀이들을 모두 지원합니다.<br>
소수점의 이동으로 인한 차이 계산, 원의 넓이 계산, 몇 가지의 숫자로 만들 수 있는 조합/순열 등등 수많은 풀이들에 대하여 학습되어있습니다.<br>
또한 한국어의 특성에 걸맞게 기존 Graph2Tree가 제시된 논문의 Quantity Cell을 결정하는 방식을 수정하여 한국어에 더 잘 맞는 모델을 구축하였습니다.<br>
